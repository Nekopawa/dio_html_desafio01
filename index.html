O que são Clients?

Voltar

O client é o usuário* que envia uma requisição ao servidor.

O cliente mais conhecido é o navegador.
Quando a requisição é feita por um navegador, ela é enviada por meio da internet. É aguardada uma resposta do servidor, e a comunicação é encerrada.
* Neste caso, um usuário web.

O que são Servers?

Voltar

O server executa as requisições e as devolve para o client. Nele, pode-se armazenar documentos, imagens, áudio, vídeo, etc.
Também existem servidores que armazenam programas para processar as requisições. Esse tipo de servidor é chamado de servidor web, ou web server. Ele executa linguagens como PHP, Javascript, Java e C#.

Servidor DNS

Também existem os servidores DNS, ou Domain Name System. Eles fazem a tradução do nome da URL, por exemplo dio.me, para seu respectivo endereço IP.
A sua função não é hospedar sites ou aplicações que criamos, mas sim permitir que as requisições cheguem aos locais corretos.

Como é uma estrutura básica de um HTML?

Voltar

<!DOCTYPE html>

Um documento HTML é formado inicialmente pela declaração <!DOCTYPE html>, ela indica ao navegador o tipo de documento esperado.

<html>

Então, deve-se utilizar a tag <html>, pois ela conterá todos os outros elementos HTML. Nela, é importante informar o atributo lang, que indicará o idioma da página.

<head>

A seguir, utiliza-se a tag <head>, que contém a metadata sobre o site, e seu conteúdo não é visível ao usuário.

<title>

Uma tag necessária em todo documento HTML, dentro da <head>, é a <title>, ela define o título da página.

<body>

Por fim, há a tag <body>, onde é definido todo o corpo do site e seus elementos.
Essa é a estrutura mais básica de um documento HTML:

<!DOCTYPE html>
<html>
<head>
  <title>Título da página</title>
</head>
<body>
   O conteúdo do documento...
</body>
</html>

Entendendo a diferença entre listas ordenadas e não ordenadas

Voltar

No HTML existem listas ordenadas (com sequência númerica ou alfabética) e não ordenadas (cada item é indicado por um símbolo).
Na lista ordenada utiliza-se a tag <ol>, e cada item dentro dela é determinado pela tag <li>. Sem informar nenhum atributo, a lista será numerada a partir de 1, 2, 3..., como no exemplo:

  1. Item 1
  2. Item 2
  3. Item 3
Porém, é possível fazer algumas mudanças. A tag <ol> aceita três atributos: reversed, especifica que a ordenação deve ser reversa (ex.: 9, 8, 7...); start, indica um valor para iniciar a lista; e type, demonstra o tipo de marcador para a lista.

Já na lista não ordenada, deve-se utilizar a tag <ul> e cada item dentro dela também é determinado pela tag <li>.
Exemplo.:
  • Item 1
  • Item 2
  • Item 3
Nesse caso, a customização deve ser feita via CSS.
Também é possível criar listas dentro de listas.
